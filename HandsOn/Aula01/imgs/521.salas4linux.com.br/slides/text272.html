Procedimentos de Deploy

Porém, para que esse o servidor de automação, que é o responsável por automatizar o Deploy da aplicação, saiba que alguma alteração foi feita, será necessário cadastrar uma WebHook no servidor de controle de versão (no nosso caso o Gitlab), para que ele avise o Jenkins toda vez que ocorrer um novo PUSH, ou seja, toda vez que alguém enviar uma alteração para o servidor de controle de versão.

Esse ciclo deve ser automatizado via scripts no Python. Assim, toda vez que algum desenvolvedor enviar uma alteração para o servidor de controle de versão, ele será executado e irá atualizar a aplicação que está no servidor de produção.
