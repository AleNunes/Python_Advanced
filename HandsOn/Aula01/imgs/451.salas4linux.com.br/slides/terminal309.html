Rsyslog

  • Ao configurar um arquivo de log o destino do log gerado pode variar conforme abaixo:
  • /dev/tty → O destino é uma console;
  • /var/log/arquivo.log →O destino do log é um arquivo de log;
  • @host_destino → O destino é uma máquina remota;
  • usuário → O destino é um usuário específico conectado a máquina.

Servidor: Audit

Notes:

Exemplos de uso prático dos destinos acima:

/dev/tty → mail.* /dev/tty5;

/var/log/arquivo.log → kern.emerg /var/log/messages;

@host_destino → *.* @192.168.20*.X

usuário → auth,authpriv.* root
