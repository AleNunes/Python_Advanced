Implementando Controle de Acesso ao Sistema com sudo

Servidor: Fileserver

O comando sudo:

  • O arquivo de configuração /etc/sudoers possui a seguinte sintaxe:
  • usuário(s)/grupo servidor = (usuário alternativo) comando
  • → usuário(s)/grupo: Usuário ou grupo que receberá a permissão. É possível estabelecer regras para múltiplos usuários separando-os por vírgulas, sendo que os grupos deverão ser indicados por %;
  • → servidor: Servidor sobre o qual a regra será definida, geralmente representado pelo curinga ALL para todos os servidores;
  • → usuário alternativo: Usuário com o qual será feita a autenticação;
  • → comando: Comando a ser executado, é possível estabelecer múltiplos comandos utilizando vírgulas.
