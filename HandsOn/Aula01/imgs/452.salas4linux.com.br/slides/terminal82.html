Mascaramento no Firewall:

Os servidores DNS estão se comunicando porém a regra que criamos de mascaramento no firewall faz com que o ip de origem seja mudado, dentro das subredes.

Com esta mudança o servidor DNS master irá bloquear a transferência de zona, pois apenas liberamos o ip da máquina WebServer Interno.

Para resolver este problema devemos criar uma exceção, mascarando apenas se o destino for a internet e não a rede interna.
