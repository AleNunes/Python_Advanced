  • Simulando problemas de Biblioteca:

A biblioteca libattr.so é utilizada pela maioria dos comandos no Linux.

DICA: Configurar o novo caminho da biblioteca.

Note que ao movermos uma biblioteca para um diretório desconhecido do do ld.so (Loader) o comando ls parou de funcionar.

Vamos corrigir esse problema!

Servidor: Webserver Interno

1# ls /etc/

2# which ls

3# ldd /bin/ls | grep libattr

4# mkdir /opt/lib

5# mv /lib64/libattr.so.1 /opt/lib/

6# ls /etc/


Notes:

Referenciando Bibliotecas

No exemplo acima, não foi possível executar o comando ls, pois a biblioteca compartilhada não pôde ser carregada por estar em um diretório que não é referenciado em /etc/ld.so.cache para os linkadores carregarem-na.
