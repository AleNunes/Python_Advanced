Quando trabalhamos com banco de dados, precisamos saber também fazer as cardinalidades. As mais comuns são:

  • Um para Um (1 -> 1)
  • Um para Muitos (1 -> N)
  • Muitos para Muitos (N -> N)

Lembrando que quando falamos de um relacionamento muitos para muitos, temos dois tipos de tabelas: as associativas e as associativas com atributos, que são criadas de maneiras diferentes no SQLAlchemy.
